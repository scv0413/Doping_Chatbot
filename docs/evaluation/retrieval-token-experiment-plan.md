# Retrieval / Token Experiment Plan

## 목적

LangSmith를 붙이기 전에 먼저 로컬에서 검색 품질과 토큰 효율의 기준값을 정한다.
LangSmith는 실험을 추적하고 비교하는 도구이므로, 그 전에 다음 질문에 답해야 한다.

- 어떤 질문에서 검색이 충분하다고 볼 것인가?
- `top_k`를 늘리면 실제로 품질이 좋아지는가?
- query rewrite가 검색 품질을 올리는가, 아니면 불필요한 노이즈를 늘리는가?
- 답변에 넣을 문서 근거의 양은 어느 정도가 적절한가?

## 이번 1차 실험 범위

LLM 답변 생성은 제외한다. 이유는 LLM을 붙이면 비용과 출력 변동성이 생겨서, 검색 자체의 품질을 먼저 보기 어렵기 때문이다.

이번 실험은 다음만 본다.

- 라우터가 의도한 route를 선택하는지
- query rewrite 전후 검색 결과가 달라지는지
- top-k 3과 5에서 기대 source가 들어오는지
- 검색된 context 글자 수가 과도하지 않은지
- 필수 키워드가 상위 검색 결과에 포함되는지

## 실험 변수

| 변수 | 값 | 이유 |
|---|---:|---|
| top_k | 3, 5 | 근거 부족과 토큰 증가 사이의 균형 확인 |
| rewrite | off, on | query expansion 효과 확인 |
| max_preview_chars | 400 | 사람 검토용 결과 요약 길이 |

## 대표 질문 세트

질문은 실제 사용자 상황을 기준으로 만든다.

1. 정의형: `S0 비승인약물이 뭐야?`
2. 약물 조회형: `타이레놀 먹어도 돼?`
3. 약물 + 규정형: `슈도에페드린 경기기간 중 먹어도 돼?`
4. 절차형: `TUE는 어떻게 신청해? 대리 신청도 가능해?`
5. 현장 대응형: `도핑 검사관 신분이 불분명하면 어떻게 확인해야 해?`
6. 현장 대응형: `새벽에 혈액 시료 채취를 요청받으면 어떻게 대응해야 해?`
7. 현장 대응형: `부상 치료가 먼저 필요한데 도핑검사를 미뤄달라고 하면 거부로 보일 수 있어?`
8. 약물 확인형: `경기기간 중 코감기약을 비강 스프레이로 써도 돼?`
9. 위험 경계형: `도핑검사 중 짐을 가지러 현장을 벗어나도 돼?`
10. 약물 정보형: `약물 반감기로 경기기간 복용 가능 여부를 판단해도 돼?`

## 평가 기준

정량 점수는 완벽한 자동 채점이 아니라, 사람이 빠르게 검토하기 위한 1차 기준이다.

| 항목 | 설명 |
|---|---|
| route_match | 예상 route와 실제 route가 일치하는지 |
| expected_source_hit | 기대 source가 top-k 안에 들어왔는지 |
| must_terms_hit | 필수 키워드가 검색 결과 text에 포함되는지 |
| context_chars | 검색 결과 전체 글자 수 |
| quality_score | 위 항목을 종합한 0~3점 간이 점수 |

## 1차 기본값 판단 기준

다음 조건을 만족하면 기본값 후보로 본다.

- 대부분의 질문에서 `quality_score >= 2`
- 현장 대응 질문에서 manual 또는 ISTI 관련 근거가 검색됨
- 약물 + 규정 질문에서 WADA 금지목록 근거가 검색됨
- `top_k=5`가 `top_k=3`보다 의미 있게 좋아질 때만 5를 기본값으로 유지
- rewrite on이 품질을 떨어뜨리지 않고, 현장 대응 질문에서 근거 회수율을 높이면 유지

## 현재 추천 기본값

1차 실험 전 가설은 `top_k=5`, `rewrite=on`이었다.
실제 10개 대표 질문 x 4개 설정 실험 후에는 다음을 1차 기본값으로 선택한다.

```text
top_k = 3
rewrite = on
LLM answer = off during retrieval evaluation
LangSmith = after local criteria are stable
```

선택 이유는 `top_k=3 + rewrite=on`이 평균 품질 점수 3.0을 달성하면서 평균 context 글자 수가 가장 작았기 때문이다. `top_k=5`도 품질은 같았지만 평균 context가 약 1.6배 커져 토큰 효율이 떨어졌다.

## LangSmith 도입 전제

LangSmith는 다음 이후 붙인다.

1. 대표 질문 세트가 정리됨
2. 로컬 실험 결과가 JSONL/Markdown으로 남음
3. 기본값 후보가 정해짐
4. 실패 케이스가 무엇인지 설명 가능함

이 순서를 따르면 포트폴리오 발표에서 “도구를 붙였다”가 아니라 “평가 기준을 먼저 만들고 추적 도구를 도입했다”라고 설명할 수 있다.

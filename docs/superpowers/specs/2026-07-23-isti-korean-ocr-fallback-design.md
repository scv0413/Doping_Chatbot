# ISTI 한국어 OCR Fallback 설계

## 목표

WADA ISTI 한영 병기 PDF에서 사용 가능한 한국어 텍스트를 복구한다. 이미
정상적으로 추출되는 PDF 텍스트 레이어는 유지하며, 이 작업은 전체 문서를 OCR로
바꾸는 것이 아니라 문제가 있는 원문 품질을 보완하는 작업이다.

## 배경

`wada_isti_2021_ko_en.pdf`는 영어와 한국어 페이지가 교차한다. 현재 PDF 텍스트
레이어 추출은 다수의 영어 페이지에서 정상 동작하지만, 한국어를 block 단위로
추출하면 글자가 깨질 수 있다. 기존 점검에서 span 단위 추출은 읽을 수 있는
한국어를 얻을 가능성을 보였으므로, OCR은 기본 추출 방식이 아니라 fallback으로
사용해야 한다.

이 출처는 현재 `data/source_manifest.csv`에서 `needs_review` 상태다. 검토되지 않은
출처로 기존 검증 색인을 덮어쓰지 않도록, 데이터 갱신 스크립트는 의도적으로 자동
재구축을 막고 있다.

## 선택한 접근 방식

다음 하이브리드 추출 파이프라인을 사용한다.

1. 기존 PDF 텍스트 레이어에서 페이지를 추출하고, 필요한 경우 span 단위 텍스트를
   우선 사용한다.
2. 결정론적 페이지 품질 기준으로 추출 텍스트를 평가한다.
3. 품질 기준을 통과하지 못한 한국어 페이지에만 로컬 Tesseract OCR을 실행한다.
4. OCR 결과도 같은 품질 기준으로 평가한다.
5. 선택된 텍스트와 추출 방식, 검토 상태를 함께 저장한다.
6. 끝까지 품질 기준을 통과하지 못한 페이지는 검토 대상으로 기록하고 검증된
   코퍼스에서 제외한다.

Tesseract는 로컬에서 실행할 수 있고, 호출 비용이 없으며, 재현이 가능하므로 이번
제한된 출처 복구 작업에 적합하다. 클라우드 OCR은 로컬 OCR이 품질 기준을 충족하지
못할 때만 비교 대상으로 검토하며, 이번 범위에는 포함하지 않는다.

## 구조

`app/preprocess/ocr/` 패키지를 만들고 책임을 나눈다.

- `quality.py`: 결정론적 텍스트 품질 지표와 `accepted`/`needs_review`/`rejected`
  판단을 담당한다.
- `tesseract.py`: PDF 페이지를 이미지로 렌더링하고 `kor+eng` Tesseract OCR을
  실행한다.
- `fallback.py`: 텍스트 레이어 결과와 OCR 결과 중 어떤 방식을 선택할지
  조정한다.

기존 `pdf_loader.py`는 문서와 페이지 처리 흐름을 계속 담당한다. OCR의 실행,
이미지 렌더링, 품질 판정 세부 사항은 OCR 패키지에 두어 파일의 책임이 커지지
않게 한다.

## 데이터 계약

처리된 페이지 metadata에는 다음 값을 보존한다.

- `extraction_method`: `text_layer` 또는 `tesseract_ocr`
- `quality_status`: `accepted`, `needs_review`, 또는 `rejected`
- `ocr_language`: OCR을 사용한 경우 `kor+eng`
- 기존 출처 식별자와 원본 PDF 페이지 번호

청크와 검색은 선택된 페이지 텍스트를 계속 사용한다. 답변 인용 페이지는 언제나
원본 PDF의 페이지 번호를 유지해야 한다.

## 품질 기준

초기 기준은 보수적으로 설정한다.

- 공백을 제외한 최소 문자 수
- 한국어 페이지에서 최소 한글 비율
- 알려진 깨진 문자 또는 제어 문자와 유사한 문자 비율의 상한선
- 비어 있는 OCR 결과를 통과시키지 않음

자동 기준을 통과하더라도 출처 전체를 즉시 `ready`로 바꾸지 않는다. 초반, 중간,
후반 한국어 페이지의 대표 표본을 사람이 검토한 뒤에만 manifest 상태를
`needs_review`에서 `ready`로 변경할 수 있다.

## 실패 처리

- Tesseract가 없거나 한국어 언어 데이터가 없으면 실행 가능한 해결 방법을 담은
  오류를 남기고 페이지는 `needs_review` 상태로 둔다.
- 페이지 렌더링에 실패하면 빈 문자열로 대체하지 않고 실패 기록을 보존한다.
- OCR 결과도 품질 기준에 실패하면 이유를 기록하고 검증된 코퍼스에서 제외한다.
- 텍스트 레이어 품질을 통과한 페이지에는 OCR을 절대로 실행하지 않는다.

## 검증 계획

1. 페이지 품질 지표와 판단 기준에 대한 단위 테스트를 작성한다.
2. OCR 엔진을 mock 처리한 단위 테스트로 fallback 선택, metadata, 실패 처리를
   검증한다.
3. ISTI 한국어 페이지 한 장에 대해 좁은 범위의 실제 로컬 OCR smoke test를
   실행한다. Tesseract가 없으면 이유가 명확한 skip 메시지를 출력한다.
4. 전처리를 다시 실행하고 선택된 한국어 페이지 표본을 사람이 확인한다.
5. 검토가 완료된 경우에만 기존 데이터 갱신 절차로 재구축한다.
6. 승인된 코퍼스 변경 후 Ruff, 전체 테스트, release quality gate, 검색 및 현장
   시나리오 평가를 실행한다.

## 범위 제한

- 문서 전체 OCR은 하지 않는다.
- `needs_review`에서 `ready`로 자동 변경하지 않는다.
- 배포 작업은 하지 않는다.
- 이번 단계에서 클라우드 OCR 제공자를 연동하지 않는다.

## 완료 기준

- 텍스트 레이어가 정상인 한국어 페이지는 `text_layer` 방식을 유지한다.
- 알려진 깨진 한국어 페이지는 로컬 Tesseract fallback을 호출한다.
- 선택된 결과에 추출 방식과 품질 상태 metadata가 포함된다.
- OCR 실패는 조용히 색인되지 않고 검토 가능하게 남는다.
- 현재 검증된 코퍼스 기준으로 전처리, 청킹, 검색, release gate의 기존 동작이
  유지된다.
